Postmortem on TreeTracker Join: Simple, Optimal, Fast

Date Jan 11, 2026 Tags academic / paper / my research

After five years of blood, sweat, and tears, TreeTracker Join has finally been published. As a postmortem, I document some notes I took while I was implementing the system side of the work. Then, I conclude the post with some personal thoughts and a bonus content!

Build a System in Academia

https://www.starburst.io/blog/jdbc-trino-starburst/

  • Don’t use Java. People worry about “lower bound” (have to compare with existing system)
  • what system to build (DM criticism on 弗兰克斯坦/ emulator） 1. can we afford throw-away? 2. trade-off simple design (fast development) vs. high-performance (use JDBC vs managing storage myself; system-assist algorithm design) 3. system comparison or algorithm comparison? 4. system library or system system comparing existing system is to provide a lower bound on algorithm implementation
  • exploit leeway (no parser, optional optimization, bring down Postgres performance) no code for a missing piece means no need to optimize
  • engineering experience: 1. profiling 2. no stream operation 3. use primitive type if possible 4. manual column-oriented (no row) 5. preallocating hash table size 6. high-performance standard library 7. use integer comparison instead of string comparison (echo engineering WCOJ paper) 8. better JDBC (DuckDB vs Postgres) 9. avoid repeated object creation

Departing Thoughts

This work was initiated by my previous advisor. Publication is happy news but the cost was dropping out of the PhD program. The pain was too much, both for me and my wife. If there is any advice I could give: it's important to work with someone who is truly looking out for you and not using you as some tool that will be discarded in the long run; the research topic should, at best, be second on your list of preferences when deciding whom to work with. If there is any red flag, no matter how tiny it might seem, pay attention to it; try to figure out if that's due to some misunderstanding, and if not, run fast and run far!

Note

I'm grateful to everyone who helped me during the difficult time in the past five years. I'm happy to share my personal experience working under my previous advisor, just as his former students helped me.
